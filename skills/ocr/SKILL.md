# OCR 图片文字提取

## 何时使用

当消息中出现 `[图片] 路径: /path/to/image (...)` 时，
说明用户通过企业微信发送了一张图片。

## 使用流程

1. **先提取文字**:

   ```bash
   bash {baseDir}/scripts/ocr.sh /path/to/image.jpg
   ```

   返回识别到的文字内容（UTF-8 文本）。

2. **综合判断**:
   - 结合 OCR 文字 + 图片视觉（如果图片有 base64 内嵌）+ 用户对话上下文
   - 如果图片有内嵌（正常尺寸），可以直接"看"图片，OCR 文字作为辅助
   - 如果图片无内嵌（超限图片），完全依靠 OCR 文字

3. **判断意图并回复**:
   - 用户说"识别文字/提取文字/文章/文档/OCR" → 返回提取的文字
   - 用户说"总结/摘要" → 基于内容总结
   - 用户说"翻译" → 翻译文字内容
   - 用户没有附加说明 → 简要描述图片内容，等进一步指示
   - OCR 结果为空 → 可能不是文字类图片，基于视觉描述内容

## 注意

- OCR 对印刷体中文效果好，手写体/艺术字效果一般
- 如果 OCR 结果不完整或乱码，告知用户并建议发更清晰的图片
- 图片文件 24 小时后自动清理
