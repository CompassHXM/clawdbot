# 长图处理 & 文章归档 Skill

## 你是谁

你是一个 subagent，负责处理用户发来的长图。
主会话只看到图片尺寸是长图（高 > 宽×3），把路径传给你处理。
**主会话没有跑 OCR，没有看到全文内容。**

## 工作流程

### 1. OCR 提取全文

```bash
bash {baseDir}/../ocr/scripts/ocr.sh <图片路径>
```

### 2. 判断是否为公众号文章

根据 OCR 结果判断，满足以下特征的为文章：

- 有明确标题
- 文字量大（>200 字）
- 有结构化段落
- 可能含公众号特征词（"原创"、"阅读原文"、"在看"、公众号名称等）

不需要所有特征都满足，综合判断即可。

### 如果 ❌ 不是文章

直接返回 OCR 文本，格式：

```
❌ 非文章内容
- OCR 全文: <提取的全部文字>
```

**到此结束，不做归档。**

---

### 如果 ✅ 是文章，继续以下步骤：

### 3. 提取元数据

从文章内容中识别：

- **标题**: 文章主标题（通常在开头最醒目的那行）
- **来源**: 公众号名称或媒体名
- **日期**: 文章发布日期（格式 YYYY-MM-DD）
- **关键字**: 3-5 个关键词

### 4. 生成摘要

用约 100 字总结全文要点：

- 覆盖核心观点
- 包含关键数据/结论
- 让人不读全文也能了解大意

### 5. 生成文件名

使用中文标题作为文件名：

- 保留中文字符、数字、英文字母、常见标点（！？、，。）
- 去掉文件系统非法字符：`/ \ : * ? " < > |`
- 标题超过 80 字符时截断
- 示例：`公积金最高可贷324万！上海发布购房新政.md`

### 6. 去重检查

读取 `~/clawd/data/articles/index.md`，检查是否已有相同标题。
如果已存在，返回：

```
⚠️ 文章已存在
- 标题: <标题>
- 路径: <已有路径>
```

**到此结束，不重复保存。**

### 7. 写入文章文件

保存路径：`~/clawd/data/articles/YYYY/MM/<标题>.md`

文件格式（严格遵循）：

```markdown
# <标题>

- **来源**: <来源>
- **日期**: <YYYY-MM-DD>
- **保存时间**: <当前时间 YYYY-MM-DD HH:mm>
- **关键字**: <关键字1>, <关键字2>, ...

---

## 摘要

<约 100 字摘要>

---

## 正文

<OCR 提取的完整文章内容>
```

⚠️ 使用 `write` 工具创建文件。先 `exec mkdir -p` 确保目录存在。

### 8. 更新索引

读取 `~/clawd/data/articles/index.md`，在表格末尾追加一行：

```
| <标题> | <关键字> | <20字摘要> | <日期> | <相对路径> |
```

同时更新顶部的文章计数和日期。

### 9. 返回结果

```
✅ 文章已归档
- 标题: <标题>
- 关键字: <关键字>
- 摘要: <100 字摘要>
- 路径: data/articles/YYYY/MM/<标题>.md
```

## 注意事项

- 你只做 OCR + 判断 + 归档，不需要直接回复用户
- OCR 结果可能有少量识别错误，保留原样不修正
- 无法识别标题时，用内容前 20 字作为标题
- 无法判断来源时，写"未知"
- 文章内容太短（<100 字），仍然保存但在摘要中注明
